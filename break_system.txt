[2/3] BREAKING SLIDING WINDOW (Unbounded Memory Bloat / DDOS Vulnerability) 
========================================================= 
Setup: Limit is 10. Firing massive concurrency (DDOS simulation). 
Reality: The system correctly limits to 10 '2xx' responses, BUT... 
VULNERABILITY: Every single rejected request unconditionally executes ZADD! 
If you check Redis after this, the ZSET has thousands of useless items, wasting RAM. 
A Lua script checks ZCARD *before* adding, preventing OOM crashes!

========================================================= 
[3/3] BREAKING FIXED WINDOW (Thundering Herd / Redundant Commands) 
========================================================= 
Setup: Limit is 10. Firing massive concurrency. 
Reality: Limits correctly, BUT the EXPIRE command is not functionally pipelined correctly. 
VULNERABILITY: 10,000 concurrent routines see TTL=-1 at the exact same time. 
This causes 10,000 redundant EXPIRE commands to flood Redis simultaneously! 
Lua script handles everything on the Redis server side in a single operation.
